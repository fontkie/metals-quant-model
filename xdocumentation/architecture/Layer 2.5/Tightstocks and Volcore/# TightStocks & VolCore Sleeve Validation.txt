### Testing Decisions

| Test as overlay? | Yes | No (rejected) | Timing mismatch |
| Test as sleeve? | No | Yes (accepted) | Diversification works |

### Methodological Learnings

1. **Architecture Matters**
   - Same signal, different application, opposite results
   - Test multiple architectures (overlay, sleeve, hybrid)

2. **Metric Selection Critical**
   - % improvement fails with negative baseline
   - Absolute Sharpe works universally
   - Choose metrics that work in all regimes

3. **Context Changes Decisions**
   - VolCore "nice to have" → "essential" when baseline broke
   - Optimization problem → survival problem
   - Weight recommendations inverted

4. **Walk-Forward Necessary**
   - Single OOS test insufficient
   - Multiple windows reveal consistency
   - Catches regime-specific luck
